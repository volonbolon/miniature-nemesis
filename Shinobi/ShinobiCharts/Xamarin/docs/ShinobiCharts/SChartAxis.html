SChartAxis Class

Abstract base class for a horizontal or vertical axis providing a visual scale for chart data

Syntax

[MonoTouch.Foundation.Register("SChartAxis", true)]
public abstract class SChartAxis : MonoTouch.Foundation.NSObject

See Also

SChartNumberAxis
SChartDateTimeAxis
SChartDiscontinuousNumberAxis
SChartDiscontinuousDateTimeAxis
SChartCategoryAxis
SChartLogarithmicAxis
SChartAxisStyle

Remarks

The SChartAxis is responsible for managing the coordinate space of the chart. It is the link between the set of real data in a series and the laying down of that series on a chart in a desired manner. Each series on the chart is linked to an axis and this SChartAxis is responsible for managing all of the series linked to it.

For cartesian series, it is expected that you will provide a subclass of SChartAxis each for the XAxis and YAxis of the chart. If not set, both axes will default to instances of ShinobiCharts.SChartNumberAxis.

The axis range can be set to a desired range or left to calculate its own minimum and maximum. When auto-calculating the range, it will consider all data series associated with it. You can set the desired range by passing in an ShinobiCharts.SChartRange object when constructing the axis, or by setting the SChartAxis.DefaultRange property.

The axis is also the home of the tick marks and their labels. These can be auto-calculated or set to specific values.

Documentation for this section has not yet been entered.

Remarks

For example, if your AxisRange is 1 to 9, your AnchorPoint is 0 and the CurrentMajorTickFrequency is 2, you will get tickmarks and labels at 2, 4, 6 and 8, assuming that the labels' text and the clipping settings permit. Regardless of whether a tick mark frequency has been set or automatically calculated, it must start somewhere. This value acts as the origin point for tickmarks on the axis. CalculateMajorTickFrequencyFromDiscontinuousRange Property

This property controls how the auto-tick-frequency calculations are done.

Syntax

[get: MonoTouch.Foundation.Export("calculateMajorTickFrequencyFromDiscontinuousRange")]
[set: MonoTouch.Foundation.Export("setCalculateMajorTickFrequencyFromDiscontinuousRange:")]
public virtual bool CalculateMajorTickFrequencyFromDiscontinuousRange { get; set; }

Value

Documentation for this section has not yet been entered.

Remarks

Normally, the axis will use the full range of data to calculate how often to place a tick-mark, however if the range has discontinuities, this option allows the axis to exclude the discontinuities in its calculations of the range over which the tick marks are added.

For example - you have a year range, but are excluding 7 months using discontinuous periods. If this property is false, the tick labels will be added as if the range is one year. If this property is true, the tick labels will be added as if the range is 5 months (12 - 7).

If true, the axis will exclude discontinuities when calculating the range.

Defaults to false.

Attempts to set the current visible range SChartAxis.AxisRange to a range with the given minimum and maximum values.

Parameters

The minimum value to be displayed, in data terms.

Remarks

Given any restrictions on setting the range, such as SChartAxis.AllowPanningOutOfMaxRange etc, this method will attempt to set the current axis range.

myDateTimeAxis.SetRange(NSDate.FromTimeIntervalSince1970(86400.0*365*43.5), NSDate.FromTimeIntervalSince1970(86400.0*365*44));

Documentation for this section has not yet been entered.

Remarks

  • SChartTickLabelClippingMode.TicksAndLabelsPersist - Keep both the label and tick visible for as long as possible, allowing the label to overlap with adjacent axes.
  • SChartTickLabelClippingMode.TicksPersist - Keep the tick mark visible for as long as possible and clip the label as soon as it reaches the edge of its view.
  • SChartTickLabelClippingMode.NeitherPersist - Keep the tick mark visible as long as the tick label is visible - as soon as the tick label reaches its bounds, and is clipped, clip the tick mark too.
