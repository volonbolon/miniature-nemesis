SChartDataSource Class

Abstract base class for your class that links your data to your chart

Syntax

[MonoTouch.Foundation.Model]
[MonoTouch.Foundation.Register("SChartDatasource", true)]
public abstract class SChartDataSource : MonoTouch.Foundation.NSObject

Remarks

You must create a SChartDataSource-derived object and assign it to the Shinobichart.DataSource property on your chart.

The data source provides the chart with the information it needs to construct the chart object. In general, styling and event response is handled by objects subclassing ShinobiCharts.SChartDelegate - the data source has minimal impact on the look and feel of the chart.

The required methods provide the chart with information about all of the series to be displayed on that chart - and the data for those series. The relevant chart object is always returned - to support the option of using a single object as the data source for multiple charts.

GetDataPoint Method

Returns the object that represents the data point at the given index in the specified chart series.

Syntax

[MonoTouch.Foundation.Export("sChart:dataPointAtIndex:forSeriesAtIndex:")]
public abstract SChartData GetDataPoint (ShinobiChart chart, int dataIndex, int dataSeriesIndex)

Parameters

chart
The chart on which the data point will be drawn.
dataIndex
The index of the data point in the chart series which contains it.
dataSeriesIndex
The index of the series which contains the data point, in the chart.

Returns

Documentation for this section has not yet been entered.

Remarks

The number of points in each series is set by the SChartDataSource.GetNumberOfDataPoints(ShinobiChart, int) method.

The data point can be any object that implements the SChartData protocol. We provide a set of data point classes which can be used. These are:

  • ShinobiCharts.SChartDataPoint - used for simple cartesian data points.
  • ShinobiCharts.SChartRadialDataPoint - used for radial data points.
  • ShinobiCharts.SChartMultiYDataPoint - used for cartesian data points with multiple y values.
  • ShinobiCharts.SChartMultiXDataPoint - used for cartesian data points with multiple x values.

GetInnerRadius Method

Returns the inner radius of the given data point, in points.

Parameters

dataIndex

dataSeriesIndex
The index of the series in the chart which contains the data point.

Returns

Documentation for this section has not yet been entered.

Remarks

By default, data points on the chart are displayed as two circles, one inside the other. The inner radius of a data point defines the radius of the inner circle in its representation on the chart. If you implement this method, and it returns a positive, non-zero value, that radius is used for the inner circle on the given data point in the specified series on the chart.

GetSeries Method

Returns the SChartSeries object at the given index in the specified chart.

Syntax

[MonoTouch.Foundation.Export("sChart:seriesAtIndex:")]
public abstract SChartSeries GetSeries (ShinobiChart chart, int dataSeriesIndex)

Parameters

chart
The chart on which the series will be displayed.
dataSeriesIndex
The index of the given series in the chart.

Returns

Documentation for this section has not yet been entered.

Remarks

The number of SChartSeries in the chart are specified by the SChartDataSource.GetNumberOfSeries(ShinobiChart) method.

The order is important for some series layouts - lower index series are placed on the chart first and will be behind subsequent series.

One of the concrete subclasses of SChartSeries will be returned here. The available types are:

  • ShinobiCharts.SChartBarSeries
  • ShinobiCharts.SChartColumnSeries
  • ShinobiCharts.SChartLineSeries
  • ShinobiCharts.SChartStepLineSeries
  • ShinobiCharts.SChartBandSeries
  • ShinobiCharts.SChartScatterSeries
  • ShinobiCharts.SChartCandlestickSeries
  • ShinobiCharts.SChartOHLCSeries
  • ShinobiCharts.SChartDonutSeries
  • ShinobiCharts.SChartPieSeries
When you reload data on a chart, SChartSeries.ResetForReuse() is called on its chart series. If you are persisting your chart series between data reloads, you should take this into account. SChartSeries.ResetForReuse() clears the data points on the chart series, while leaving styling, animation and other series properties intact.

GetXAxis Method

Returns the x axis for this series on the chart.

Syntax

[MonoTouch.Foundation.Export("sChart:xAxisForSeriesAtIndex:")]
protected virtual SChartAxis GetXAxis (ShinobiChart chart, int dataSeriesIndex)

Parameters

chart
The chart containing the axis.
dataSeriesIndex
The index of the chart series within the chart.

Returns

Documentation for this section has not yet been entered.

Remarks

If this method is implemented the chart will look here for which axis to use when representing this series, otherwise this series will default to using the primary xaxis on the chart.

Hint: This is only needed on charts with multiple axes.

Note: The SChartAxis returned should be referenced from an existing chart axis and not a new object. eg: return chart.XAxis

To specify an axis for some series and not for others, returning null for a series index will revert to the default behaviour for that series.
